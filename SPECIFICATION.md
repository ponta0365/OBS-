# OBS字幕メーカー 技術仕様書 (Specification)

## 1. 概要
本ソフトウェア「OBS字幕メーカー」は、OBS Studioの録画機能とリアルタイムに連動し、録画中のキー操作に基づいてタイムスタンプ（字幕データおよびチャプターデータ）を収集し、録画終了時に動画ファイルと同一フォルダに同名の字幕ファイル（SRT形式）を自動生成するWindowsデスクトップアプリケーションです。

---

## 2. システム要件と技術スタック
- **対応OS**: Windows 10 / 11
- **開発言語**: Python 3.10以上
- **主要ライブラリ**:
  - **GUI**: `customtkinter` (ダークテーマ対応のモダンUI)
  - **OBS通信**: `obsws-python` (OBS WebSocket 5.x プロトコル対応)
  - **キーフック**: `keyboard` (グローバルホットキーのフック)
  - **通知**: `win11toast` (Windows 10/11ネイティブ通知の送信)
  - **テスト/ブラウザ検証**: `playwright`

## 4. 機能仕様

### 4.1. OBS接続と制御
1. **WebSocket接続**: ユーザーが指定したホスト、ポート、パスワード情報を用いて OBS Studio の WebSocket サーバーに接続します。
2. **自動起動**: `Start OBS` ボタン押下時、指定された実行ファイルパス（`obs64.exe`）から OBS Studio をバックグラウンド起動し、接続ができるまで自動試行します。
3. **プロファイル/シーン設定の同期**:
   - 接続成功時、OBSから現在有効なプロファイルおよびシーンコレクション一覧を取得し、GUIのプルダウンに反映します。
   - `Apply to OBS` ボタン、あるいは録画開始時に、選択された設定をOBSにリアルタイムで反映します。

### 4.2. 録画ディレクトリの同期 (最新仕様)
本アプリからOBSの動画保存先ディレクトリを直接制御できます。
1. **基準フォルダ (Base Folder)**: 録画ファイルを整理するルートフォルダを指定します。
2. **出力フォルダ (Output Folder)**: 実際に動画が保存されるフォルダです。
3. **日付フォルダ自動作成**:
   - `Create Date Folder` ボタンを押すと、基準フォルダの下に今日の日付（`YYYY-MM-DD`）のフォルダを自動作成し、出力フォルダ欄にセットします。
4. **リアルタイムOBS反映**:
   - 録画開始時、または設定保存時に、OBS WebSocket v5.3+ の `SetRecordDirectory` API （リクエスト名: `SetRecordDirectory`, 引数: `recordDirectory`）を用いて、**OBS側の録画保存先パスを即座に書き換え**ます。
   - 互換性確保のため、古いOBSバージョン向けには `set_profile_parameter`（カテゴリ: `SimpleOutput`/`AdvOut`, パラメータ名: `FilePath`/`RecFilePath`）を用いて設定値を上書きするフォールバック処理を行います。

### 4.3. グローバルホットキーと字幕入力仕様（カスタマイズ・競合対策対応）
録画開始から終了までの経過時間（ミリ秒単位）を基準に、設定された任意のキーの組み合わせ（初期値は以下）でタイムスタンプとテキストをマーカーとして記録します。

| 設定項目名 | 初期ホットキー | 動作内容 |
| :--- | :--- | :--- |
| **Start/Stop Rec Hotkey** | **Ctrl + Alt + R** | 録画の開始 / 停止 をトグル切り替えします。 |
| **Record Subtitle Hotkey** | **Ctrl + Alt + T** | 設定画面で事前登録した「設定字幕」テキストを、押下した瞬間のタイムスタンプで記録します。 |
| **Open Input Window Hotkey** | **Alt + G** | 自由入力用の「字幕入力小窓」を表示します（後述）。 |
| **Add Chapter Hotkey** | **Alt + C** | 自動連番のチャプター（例: 「チャプター 1」「チャプター 2」...）を自動カウントアップしながら記録します。 |

#### 他アプリとの競合・動作停止への対策仕様:
1. **キーバインドの自由なカスタマイズ**:
   - 上記4種類のショートカットキーは、GUIの設定画面および `config.json` からユーザーが任意に変更可能です（例: 他のゲームや配信管理ツールと競合した場合に回避キーを設定可能）。
2. **特定のフックのみを個別に解除する安全設計 (他アプリ干渉防止)**:
   - 録画終了時などの監視停止の際、`keyboard.unhook_all()` を使ってプロセス全体のキーフックを一括解除すると、本アプリ自身の「録画開始/停止キー」が消えてしまったり、他プロセスで動いている別アプリのキーフックまで巻き添えにして無効化してしまう重大な干渉リスクがあります。
   - これを防ぐため、キー登録時に登録ハンドラを配列で保持し、監視停止時にはそれら特定のホットキーのみを `keyboard.remove_hotkey(handler)` で**個別にピンポイントで解除**するよう実装しています。これにより、共存する他のPythonアプリ等とのキー干渉を最小限に抑えます。
3. **ホットキーの再バインド機能 (Reset Hotkeys)**:
   - OSの動作などによりキー監視（フック）が外れて反応しなくなった場合、サイドバーの `Reset Hotkeys` ボタンを押すことで、すべてのフックを一度クリアし安全に再フックを行います。
4. **管理者権限の自動検知**:
   - アプリケーションが一般ユーザー権限で動作している際、アクティブウィンドウ（ゲーム等）が管理者権限で実行されているとキーフックがブロックされるWindowsのセキュリティ仕様があります。
   - 本アプリ起動時に管理者権限で実行されているかを自動検知し、未実行の場合はサイドバーに警告ラベルを表示して「管理者として実行」を促します。

#### 字幕入力小窓 (Alt + G) の詳細挙動:
- ショートカット押下で、画面最前面にテキスト入力欄だけの小さなウィンドウが表示され、即座にフォーカスが当たります。
- ユーザーが文字を入力し始めると、**「入力を開始した瞬間のタイムスタンプ」が内部で仮保持**されます。
- 文字を入力し終えて `Enter` キーを押すと、仮保持されたタイムスタンプ付きで入力文字列が確定マーカーとして保存され、テキストボックスは自動で空になります。
- 小窓を開いたまま、連続して次のテキストを入力・確定することができます（マウス操作不要）。

### 4.4. 字幕ファイル (SRT) の書き出し
1. **録画停止時の連動**:
   - `Ctrl + Alt + R` または停止ボタン押下時、OBS に録画停止コマンドを送信します。
   - OBSから返されるレスポンスより、作成された動画ファイルの絶対パス（例: `H:/OBS録画ファイル/2026-06-19/output.mp4`）を取得します。
2. **SRTファイルの配置**:
   - 取得した動画ファイル名をもとに、拡張子のみを `.srt` に書き換えたファイル（例: `output.srt`）を同一ディレクトリに生成します。
   - マーカーに記録されたタイムスタンプとテキストを、設定された表示秒数（初期値: 3.0秒〜10.0秒）に基づいてSRT形式にフォーマットして書き出します。
3. **フォールバック処理**:
   - 万が一OBSから動画パスが返されなかった場合は、アプリ内の出力フォルダ (`data/output`) に `markers_タイムスタンプ.srt` という名称で字幕ファイルのみを出力します。
